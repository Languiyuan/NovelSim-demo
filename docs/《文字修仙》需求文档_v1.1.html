一、项目概述与核心决策

1.1 项目背景

开发一款基于 Taro (React) + NestJS (TypeScript) 全栈架构的修仙题材文字小程序游戏。核心目标有二：

  1. 全栈落地：跑通跨端小程序开发、后端业务逻辑、数据库设计与云服务器部署全流程。
  2. Agent 能力集成：在游戏关键流程引入大语言模型（LLM），实现可控成本下的动态叙事生成与结构化状态变更。

1.2 核心定位

  • 类型：修仙题材的文字放置类 Roguelike 叙事游戏。
  • 特色：仅在用户在线时推进游戏时间线，避免离线结算的数值膨胀与剧情跳脱感。游戏体验如同观看一部实时演算的修仙小说。
  • 平台：微信小程序 + 支付宝小程序。

2.3 世界划分与跨界机制

第一类世界（线性推进，含承载上限）

世界承载上限境界说明
凡界化神初版主要舞台。修士可继续修炼至返虚甚至更高，但受世界压制
灵界天仙化神后通过飞升进入
仙界圣境后期版本实现

世界压制机制

当主角境界超过当前世界承载上限后，会受到天地规则压制：

  • 修为获取 ×0.2，修炼速度大幅放缓。
  • 节点频率分布动态调整：修炼增益类事件减少，瓶颈/心魔类负面事件增多。
  • 压制效应在"飞升"完成后立即解除。
压制是软性惩罚而非硬性封顶，主角依然可以缓慢成长，但小说式"卡瓶颈"叙事感会自动浮现，倒逼玩家寻求飞升路径。

跨界（飞升）双路径

路径 A：天地规则飞升（主线 / 安全）

  • 触发条件：修至当前世界承载上限的巅峰（例：化神大圆满）。
  • 触发方式：强制剧情节点，类似"渡劫"，需通过天地考核。
  • 风险：考核失败可能境界回退或重伤；不会丢失法宝/资源。
  • 奖励：稳定抵达下一界核心区域，自带"飞升者"身份加成（NPC 关系初始好感+）。

路径 B：空间通道飞升（支线 / 高风险）

  • 发现方式：在游历、探险类事件、奇遇节点、机缘节点中有概率偶遇空间通道。
  • 触发条件：境界达到当前世界承载上限即可（如刚化神便可冒险闯通道），不要求巅峰。
  • 属性：每条空间通道携带 quality（品质）与 danger（危险度），由发现节点的上下文决定。
品质目的地抵达概率资源损失受伤概率
高品（稳定）灵界核心区高（90%+）低低
中品灵界边缘中（60~80%）中中
低品（残破）随机灵界小世界 / 险地低（30~50%）高（可能丢法宝）高（可能重伤甚至陨落）
设计意图：路径 A 提供主线安全感，路径 B 提供"野路子修仙"叙事张力。低品通道有概率把主角直接抛入仙界险地（极小概率超规格收益），契合修仙小说"机缘巧合越界冒险"的常见桥段。

第二类世界

  • 地仙界：可容纳所有境界，初版不实现，作为后期特殊分支路线（可由极低品空间通道偶发抵达）。

跨界生成流程

无论路径 A 还是路径 B，最终都收口到统一的两步生成：

  1. 调用 Agent3 生成新世界大纲（受到 quality/targetWorldHint 影响，决定登场区域）。
  2. 调用 Agent1 在新世界框架内生成过渡故事（路径 B 会强调"狼狈/迷茫"叙事基调）。

三、战力体系

3.1 核心属性与软属性

核心属性（基础三维）

属性字段名说明
精jing肉体强度，影响生命与防御
气qi灵力储量，影响法术威力与持续
神shen神识强度，影响感知与神通威力

派生战斗值（DerivedStats）

字段含义计算公式（示例）
hp气血上限jing × 10 + 境界系数
mp灵力上限qi × 8 + 境界系数
sp神识上限shen × 6 + 境界系数
基础三维 jing/qi/shen 仅由突破、奇遇、丹药等长期成长改变；派生值 hp/mp/sp 在战斗与节点中实时变化，由后端按公式计算，不直接持久化基础三维以外的字段。

软属性（不直接影响战力，影响事件概率）

属性作用

四、游戏流程设计

4.1 表现形式

  • 纯文字展示。
  • 界面上方：主角各项战力属性数据面板。
  • 界面上方：主角人生历程叙事流（文字滚动展示）。
  • 视觉风格：深色底色 + 古籍宣纸纹理，字体使用 霞鹜文楷 或系统默认衬线体。
  • 操作：游戏主要在后台自动运行，玩家仅需在出现蓝色高亮选项时点击决策。

4.2 游戏生命周期

转世投胎→ 初始属性生成→ 2~3 个引导事件→ 按部就班修仙→ 各境界节点触发→ 玩家选择→ 结局

结局类型：通关（大乘）/ 战死 / 寿尽。

4.3 叙事时间粒度

境界时间单位每段节点数每节点寿命消耗
练气 / 筑基周3 ~ 4 个约 7 天
金丹 / 元婴月4 ~ 5 个约 30 天
化神 / 返虚年4 ~ 5 个约 1 年
渡劫 / 大乘十年3 ~ 4 个约 10 年
  • 寿命计算：初始寿命由属性生成（气属性越高初始寿命越长），每个节点消耗固定寿命，随境界递增。寿命耗尽触发结局。
  • 时间推进规则：仅在线推进。用户退出小程序，故事世界静止。

4.4 事件正负向比例（随境界动态调整）

境界段正向比例设计意图
练气 ~ 筑基85%新手友好，建立代入感
金丹 ~ 元婴75%开始出现危机感
化神 ~ 返虚65%高风险高回报阶段
渡劫 ~ 大乘55%冲关阶段，紧张刺激
气运软属性在此基础上做加减修正，高气运角色正向比例更高。

4.5 流式输出与预生成

  • 叙事文本采用流式输出，提升阅读体验。
  • 选项直接显示（不等待流式完成）。
  • 预生成机制：用户在选择时，每个选项对应的后续内容已经预先生成并缓存，选择后可直接展示。
  • 预生成内容持久化存储，用户回溯时可复用。

5.2 战斗节点 BATTLE 核心

子类型触发方式可选行动
切磋 DUEL同阶道友约战接受 / 婉拒
遭遇 ENCOUNTER野外遭遇妖兽或敌修迎战 / 逃跑 / 谈判
死战 DEADLY仇敌或强敌追杀死战 / 舍命逃脱
守护 PROTECT需保护他人挺身 / 置身事外

5.3 奇遇节点 WONDER 核心

触发概率较低，收益极高，可改变人生走向。

子类型示例
传承奇遇发现上古洞府，获得前辈毕生修炼心得
灵宝奇遇天降异象，法宝主动认主
天材奇遇发现万年灵药，服下可直接突破瓶颈
异变奇遇修炼中突发异变，可能大成也可能走火入魔
空间奇遇险地深处发现一道扭曲空间裂缝，可探查为空间通道（潜在飞升路径 B）

5.4 机缘节点 KARMA 核心

与 NPC 或世界剧情强关联，故事 Agent 预设的"命运锚点"。

子类型示例
师门机缘掌门闭关前单独召见，传下镇派神通
宿敌机缘与宿敌命运再度交织，旧恨新仇一并清算
道侣机缘与结伴多年的道侣情愫渐生
预言机缘算命老者为主角批命，预言某事将发生
飞升机缘故人/前辈赠送残破空间通道坐标，可作为飞升路径 B 入口

5.5 闭关节点 RETREAT 扩展

玩家主动触发，不占用常规节点配额。

  • 短期闭关（1 个时间单位）：稳定修为提升，风险低。
  • 长期闭关（3 个时间单位）：可能错过机缘，也可能躲过劫难。
  • 特殊触发：气运高时闭关可能顿悟突破。

5.6 宗门/势力节点 FACTION 扩展

子类型示例
宗门任务宗门委派任务，完成获贡献点换资源
势力冲突所在势力与他派摩擦，主角被卷入
晋升考核宗门内部排名考核，影响地位

5.7 天劫节点 TRIBULATION 扩展

特定境界突破时强制触发，高风险高收益。天劫节点为额外触发节点，不计入当段常规节点配额。

触发时机：金丹 / 元婴 / 化神 / 渡劫 / 大乘 突破时必触发。

选项条件成功率
万全准备后渡劫消耗资源/法宝高
强行渡劫无需资源低
借助外力渡劫需 NPC 关系或特定法宝中高

结果：渡劫成功 / 失败（失败不一定死，可能境界回退）。

5.8 因果节点 CAUSALITY 扩展

基于玩家历史选择的延迟反馈，体现"选择真的有影响"。

  • 杀死落败对手 → N 个节点后触发"死者门人寻仇"。
  • 帮助某 NPC → 该 NPC 危急时刻出手相救。
  • 由结构化状态管理记录"因果债"，系统自动插入，不占用常规节点配额。

额外节点不计入当段配额，但仍记录到 node 表，并计入"已用节点序号"，因此会消耗寿元。

世界压制下的修正

当主角进入"超载境界"（如凡界化神后继续修返虚），表 A 中"事件"列内部的子类型分布会自动调整：

  • "修炼增益类事件"权重 ×0.3（如福地、论道）。
  • "瓶颈/心魔类负面事件"权重 ×2.0。
  • 触发空间通道相关奇遇 / 飞升机缘的概率显著提升。

六、Agent 体系设计

6.1 混合式架构

┌─────────────────────────────────────────────────────┐ │ 校验层 │ │ Agent3 — 世界大纲 Agent │ │ 职责：生成世界框架 + 校验生成内容一致性 │ └──────────────────────┬──────────────────────────────┘ ↓ 校验通过 ┌─────────────────────────────────────────────────────┐ │ 生成层 │ │ Agent1 — 故事生成 Agent │ │ 职责：转世故事、NPC 列表、命运锚点 │ └──────────────────────┬──────────────────────────────┘ ↓ ┌─────────────────────────────────────────────────────┐ │ 生成层 │ │ Agent2 — 节点生成 Agent │ │ 职责：节点内容、玩家选项、状态变更 │ └─────────────────────────────────────────────────────┘

6.2 Agent3 — 世界大纲 Agent（校验层）

触发时机：新游戏开始时，或主角进入新世界时。

输出内容

  • 世界名称与基础设定（地理、历史、当前格局）。
  • 主要修仙势力（门派 / 散修联盟 / 魔道等）。
  • 世界级别的重大事件脉络。
  • 该世界的特殊规则（如"此界灵气匮乏，修炼速度减半"）。

校验职责

  • 校验 Agent1 / Agent2 生成内容是否符合当前世界规则。
  • 过滤不符合当前境界的节点内容（如练气期出现仙界资源）。
  • 校验失败则要求重新生成，形成闭环。

6.3 Agent1 — 故事生成 Agent（生成层）

触发时机：新游戏开始时（在 Agent3 之后），或世界迁移时。

输出内容

  • 主角出身背景（家族 / 门派 / 散修）。
  • 核心 NPC 列表（师父 / 道侣 / 宿敌 / 挚友，含性格与关系）。
  • 主角命运宏观走向（松散大纲，不影响玩家选择自由度）。
  • 预设若干"命运锚点"（机缘节点的原材料）。

6.4 Agent2 — 节点生成 Agent（生成层）

触发时机：每个节点结束后，预生成接下来 2~3 个节点（滚动更新）。

输入上下文

  • Agent3 世界大纲摘要。
  • Agent1 故事脉络摘要。
  • 当前结构化状态（主角属性、持有法宝、NPC 关系、因果债记录）。
  • 最近 N 个节点的历史摘要。

输出内容

  • 节点类型与子类型。
  • 节点叙事文本（200~400 字）。
  • 玩家可选项（2~3 个，带简短说明）。
  • 各选项对结构化状态的预期影响（JSON 格式）。

边界约束

  • 配置"境界白名单"，限制生成内容的境界范围。
  • 后端增加一层校验，过滤矛盾内容。

6.5 一致性保障方案

6.5.1 结构化状态管理（存数据库，记录"事实"）

{
  "baseAttrs": { "jing": 90, "qi": 120, "shen": 80 },
  "derived":   { "hp": 950, "mp": 1020, "sp": 520 },
  "character": { "realm": "金丹", "exp": 4200 },
  "inventory": ["青竹剑（法器）", "固元丹×3"],
  "skills": ["御剑术", "火球咒"],
  "npc_relations": {
    "师父李玄真": { "status": "在世", "affinity": 85 },
    "宿敌王霸天": { "status": "在世", "affinity": -60 }
  },
  "karma_debts": [
    { "event": "杀死落败的陈刚", "triggered": false, "delay_nodes": 5 }
  ],
  "world_flags": ["已开启东海秘境", "魔道入侵已开始"]
}

6.5.2 故事摘要压缩（存数据库，记录"叙事氛围"）

  • 每 5 个节点由 Agent 自动生成一次摘要，替代原始文本输入上下文。
  • 保留关键转折点的原始文本片段。
  • 近几次对话保留，更远的通过摘要压缩，避免上下文过长导致重复/遗忘。

6.5.3 Agent 输出约束

  • Agent 必须返回 JSON 格式的指令（如 { "action": "add_exp", "value": 500 }）。
  • 严禁 Agent 直接修改数据库，所有状态变更由后端逻辑层执行。

方法路径说明请求参数返回数据

方法路径说明请求参数返回数据

battle-resolve 与 choose 的关系：choose 为通用选择接口，适用于事件/奇遇/机缘等节点；战斗节点先走 choose 选择行动（迎战/逃跑等），再调用 battle-resolve 进行战斗判定与结果生成。两步分离便于前端展示战斗过程。

修为达到阈值后自动触发突破节点。金丹/元婴/化神/渡劫/大乘 突破时额外触发天劫节点，玩家选择渡劫方式，概率判定成败。

方法路径说明请求参数返回数据
POST /api/game/breakthrough 触发境界突破判定 { saveId } { success, newRealm, narrative, needTribulation }
POST /api/game/tribulation 天劫节点（选择渡劫方式 + 结算） { saveId, method: "prepared" | "force" | "external" } { result: "success" | "fail", narrative, stateChanges, realmChange? }
7
飞升 / 跨界（双路径）

玩家可通过两种路径离开当前世界：
路径 A 天地规则飞升：境界达到当前世界承载上限的巅峰（如化神大圆满），触发强制剧情节点，安全度高。
路径 B 空间通道闯关：在游历/奇遇/机缘节点中发现空间通道后主动闯入，境界达到承载上限即可，但有概率受伤、丢宝、误入险地。

方法路径说明请求参数返回数据
POST /api/game/ascend-natural 路径 A：天地规则飞升判定 { saveId } { result: "success" | "fail", narrative, stateChanges, worldId? }
POST /api/game/ascend-channel 路径 B：空间通道闯入 { saveId, channelId } { result, narrative, stateChanges, lostProps[], injuries, worldId? }
POST /api/game/migrate-world 飞升成功后生成新世界大纲（Agent3，内部调用） { saveId, channelQuality? } { worldId, worldName, history, factions[], rules[] }
POST /api/game/migrate-story 飞升成功后生成过渡故事（Agent1，内部调用） { saveId, worldId, viaPath: "natural" | "channel" } { transitionNarrative, npcs[] }
关于空间通道的发现：通道由 Agent2 在生成游历/奇遇/机缘节点时按概率随机植入，发现后写入 game_state.state.space_channels。玩家可在任意时刻调用 /api/game/ascend-channel 主动闯入。
8
结局判定

每次状态变更后自动检测结局条件（寿尽/战死/通关）。触发结局时由 Agent 生成结局叙事文本，计算星级评价，存档转为"已完结"状态。

方法路径说明请求参数返回数据
POST /api/game/ending 结局判定与叙事生成 { saveId } { type: "death" | "age" | "victory", narrative, rating, stats }

7.6 边界规则与兜底策略

主题规则
重投限制 后端基于 Redis Key roll:{userId} 计数，TTL 24h。每次 /api/game/roll-character 调用 +1，达到 3 次后强制返回上一次结果，直至 24h 后或确认创角后清零。
仅在线推进 SSE 心跳 30s 一次。客户端断开后保留 60s 容忍窗口（处理切后台/网络抖动），超时则冻结时间线。重连后从最后一个推进点继续。
多端互斥 同一 saveId 同时只允许一个 SSE 连接。后开者顶替先开者，旧连接收到 event: kicked 后断开。前端提示"该存档已在其他设备登录"。
Agent 失败兜底 1. JSON 解析失败：自动重试 2 次（带"严格 JSON"提示词）；
2. 仍失败：降级为 mock 模板（按当前境界返回保底节点）；
3. Agent3 校验失败：最多重新生成 2 次，仍失败则放行并打"待校验"标记。
战斗节点 choose 行为 战斗节点的 /api/game/choose 仅回写 chosenIndex，不产生 stateChanges，narrative 字段返回过渡描述。真正的状态变更与战斗描述由 /api/game/battle-resolve 产生。
预生成节点 所有预生成的兄弟分支节点都落库（node.is_pregenerated=1），同一决策点的兄弟节点共享 branch_group_id。玩家选择后未选分支保留供后期回溯使用，不删除。
飞升路径 B 风险结算 /api/game/ascend-channel 按 quality + danger 综合判定结果：成功抵达 / 受伤抵达 / 失踪虚空（按 lostNodes 消耗寿元后回到原世界）/ 当场陨落。所有结果均产生叙事文本与 stateChanges。
请求频控 所有写接口（含 choose、ascend-* 等）在网关层按 userId + saveId 限流：1s 1 次。SSE 同时连接数全局限 1 万，按用户 token 复用。

prop 法宝/神通 id, save_id, name, prop_type(weapon/skill), level, affixes(JSON), stat_bonus(JSON), obtained_node_id, equipped
game_state 结构化状态 id, save_id, state(JSON), summary, summary_version, updated_at
space_channel 空间通道（飞升路径 B） id, save_id, quality, danger, target_world_hint, discovered_node_id, used, created_at
动态结构化状态（karma_debts、world_flags、npc_relations 等）使用 game_state.state 的 JSON 字段存储，避免频繁加列。

8.4 数据模型定义

Player 实体

interface Player {
  id: number;
  saveId: number;
  name: string;
  realm: string;        // 当前境界
  exp: number;          // 当前修为值
  jing: number;         // 精（基础三维）
  qi: number;           // 气（基础三维）
  shen: number;         // 神（基础三维）
  talent: number;       // 资质（软属性）
  wisdom: number;       // 悟性（软属性）
  luck: number;         // 气运（软属性）
  age: number;          // 当前年龄
  maxAge: number;       // 寿元上限
}

interface DerivedStats {
  hp: number;           // 由 jing 派生
  mp: number;           // 由 qi 派生
  sp: number;           // 由 shen 派生
}

Node 实体

interface GameNode {
  id: string;
  saveId: number;
  realm: string;              // 触发时境界
  nodeType: 'EVENT' | 'BATTLE' | 'WONDER' | 'KARMA'
          | 'RETREAT' | 'FACTION' | 'TRIBULATION' | 'CAUSALITY';
  subType: string;            // 子类型
  text: string;               // 叙事文本
  choices: Choice[];           // 选项列表
  chosenIndex?: number;       // 玩家选择
  stateChanges?: object;      // 状态变更
  createdAt: Date;
}

interface Choice {
  text: string;               // 选项文案
  hint?: string;              // 选项提示
  expectedEffects?: object;   // 预期影响（Agent 输出）
}

GameState 实体

interface GameState {
  id: number;
  saveId: number;
  state: {
    baseAttrs: { jing: number; qi: number; shen: number };
    derived:   { hp: number; mp: number; sp: number };
    character: { realm: string; exp: number };
    inventory: string[];
    skills: string[];
    npc_relations: Record<string, { status: string; affinity: number }>;
    karma_debts: { event: string; triggered: boolean; delay_nodes: number }[];
    world_flags: string[];
    space_channels: SpaceChannel[];   // 已发现的空间通道
  };
  summary: string;            // 最近的故事摘要
  summaryVersion: number;     // 摘要版本号（每5个节点更新一次）
  updatedAt: Date;
}

interface SpaceChannel {
  id: string;
  quality: 'high' | 'mid' | 'low';   // 通道品质
  danger: number;                    // 危险度 0~100
  targetWorldHint: string;           // 暗示目的地（"灵界核心区"等）
  discoveredNodeId: string;          // 来源节点
  used: boolean;
}
