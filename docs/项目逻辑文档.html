1. 项目概述

文字修仙 是一款轻量级 AI 叙事游戏 Demo。世界观固定为凡界修仙背景， 主角固定为练气中期散修李青云。玩家通过"继续"按钮推进剧情， 在关键节点做出选择，最终走向飞升、陨落或寿尽三种结局之一。

核心游戏循环

开始游戏 → AI 生成剧情 → 点击继续 → 触发决策节点 → 玩家选择 → AI 根据选择续写 → 走向结局

设计理念

极简架构：仅 4 个 API 接口、2 张数据表。摒弃数值系统与复杂 Agent，专注于叙事体验。

5. 前端逻辑

5.1 状态管理（useGameState）

全局状态通过 useReducer 管理，集中在 useGameState.ts Hook 中。 所有 API 调用均封装为 Hook 方法，组件只调用 Hook，不直接操作 api.ts。

GameState 结构

字段类型说明
phase'idle' | 'playing' | 'ended'游戏阶段（控制页面渲染）
characterGameCharacter | null当前角色信息
worldIntrostring世界观介绍文本
storyPhasestring当前剧情阶段（intro/early/mid/late）
narrativeHistoryArray叙事历史列表（用于渲染和传递给 API）
currentNodeGameNode | null当前待选择节点，null 时显示继续按钮
endingGameEnding | null结局数据，触发时渲染结局页
isLoadingboolean加载状态（禁用操作）
isTypingboolean打字机动画进行中（禁用操作）

Action 类型

GameReducer 状态转换
stateDiagram-v2 [*] --> idle: 初始状态 idle --> idle: LOADING_START idle --> playing: GAME_STARTED playing --> playing: NARRATIVE_RECEIVED playing --> playing: NODE_TRIGGERED playing --> playing: TYPING_COMPLETE playing --> playing: SET_PHASE playing --> ended: ENDING_REACHED ended --> idle: RESET playing --> idle: RESET

5.3 组件说明

StatusBar

展示角色名（李青云）、境界（练气中期）、当前剧情阶段标签。

NarrativeArea

遍历 narrativeHistory 渲染所有叙事段落，最新段落使用 TypewriterText 组件呈现打字机效果。

ActionPanel

无节点时渲染"继续"按钮触发 continueStory()；有节点时渲染多个 ChoiceButton，点击触发 makeChoice(nodeId, index)。

EndingScreen

展示结局类型（飞升/陨落/寿尽）、结局叙事和一句话命运总结，提供"重新修行"按钮调用 resetGame()。

TypewriterText

逐字符输出文本，使用 useTypewriter Hook 控制节奏，完成后回调 onTypingComplete。

LoadingIndicator

API 请求期间 isLoading=true 时显示的加载动画。

6.3 剧情阶段推进逻辑

advanceStoryPhase() 根据当前会话的 StoryNode 总数自动判断阶段：

intro · 0–2 节点 early · 3–5 节点 mid · 6–8 节点 late · 9+ 节点 ending · AI 判断触发
阶段StoryNode 数AI 写作指导
intro≤ 2故事刚开始，铺垫世界观和主角处境，节奏舒缓
early3 – 5主角开始遇到机遇与挑战，逐步展开
mid6 – 8冲突加剧，重大抉择出现，节奏加快
late≥ 9高潮迭起，命运抉择，可以考虑走向结局

7. AI 层

7.1 Provider 设计

AI 层采用策略模式，通过 AI_PROVIDER 环境变量切换实现：

MockProvider（默认）

预设固定 JSON 响应，无需真实 AI API，用于本地开发和测试。通过 resetState() 重置调用状态。

OpenAiCompatibleProvider

通过 axios 调用 OpenAI 兼容接口（支持任何兼容 API）。需配置 AI_API_URL、AI_API_KEY、AI_MODEL。

Provider 接口 所有 Provider 实现 AiProvider 接口的 generateCompletion(messages, options) 方法，返回原始 JSON 字符串，由 AiService 统一解析。

10. 节点类型 & 结局类型

决策节点类型

类型名称说明示例
event事件日常修仙生活，概率适中坊市交易、与道友切磋
battle战斗遭遇对手，需选择策略遭遇妖兽、门派冲突
wonder奇遇低概率高收益事件发现前人洞府、奇珍异宝
fate机缘命运性转折点灵根觉醒、传承降临

触发规律

节点触发频率：AI 被指导每 2–4 段叙事触发一次决策节点，每个节点提供 2–3 个选项，每个选项附带简短的提示文字（hint）。

结局类型

飞升 · ascension

突破大乘境，引发天劫，渡劫成功，白日飞升，脱离凡界。修仙之路最圆满结局。

陨落 · death

激烈战斗中落败，元神俱灭，身死道消。修行路上未走到最后。

寿尽 · lifespan

修行数百年，未能突破瓶颈，大限将至时安然坐化，魂归天地。
